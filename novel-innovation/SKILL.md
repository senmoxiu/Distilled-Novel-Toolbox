# 创新、反套路与融合引擎

## Purpose

提供网文创新的方法论与工具库，帮助作者在套路中找到新意，在融合中创造独特，在AI辅助中保持人性。

## When to Use

- 用户想要创新写法或反套路
- 用户想要融合多种题材
- 用户想要了解最新网文趋势
- 用户想要规避AI写作的常见陷阱

## Instructions

### 反套路设计库（30+种）

#### 叙事反套路
1. **反派视角**：以反派为主角
2. **配角逆袭**：配角才是真正主角
3. **时间倒流**：从结局开始叙事
4. **多重视角**：同一事件不同视角
5. **不可靠叙述者**：叙述者本身在说谎

#### 设定反套路
6. **弱化金手指**：金手指有严重副作用
7. **反转身份**：主角其实是反派
8. **世界是假的**：世界本身是某种构造
9. **系统是敌人**：金手指系统有自己目的
10. **升级是陷阱**：越强越接近危险

#### 角色反套路
11. **反派有理**：反派的目标其实更合理
12. **主角有缺陷**：主角的缺陷导致真正后果
13. **配角超越**：配角在某些方面超越主角
14. **导师背叛**：最信任的导师是敌人
15. **宿敌联手**：敌人变成最重要的盟友

#### 剧情反套路
16. **赢了但输了**：表面胜利实际损失更大
17. **输了但赢了**：表面失败实际获得更多
18. **预期违背**：读者预期A结果得到B
19. **因果倒置**：原因其实是结果
20. **循环叙事**：故事回到起点但有新理解

#### 情感反套路
21. **CP拆散**：看似天造地设的CP没有在一起
22. **仇人变亲人**：深仇大恨后理解对方
23. **英雄变恶龙**：屠龙者终成恶龙
24. **牺牲无意义**：牺牲并没有换来预期结果
25. **平凡最珍贵**：追求非凡后回归平凡

#### 类型反套路
26. **修仙是骗局**：修仙体系本身有问题
27. **系统是牢笼**：系统在控制宿主
28. **重生是诅咒**：重生不是福利而是惩罚
29. **穿越是实验**：穿越背后有操控者
30. **末世是净化**：末世不是灾难而是重生

### 2026网文趋势关键词

1. **无CP大女主**：独立成长，不依附男性
2. **发疯文学**：情绪极端化、行为不可预测
3. **考据式穿越**：严谨的历史/科学知识运用
4. **人性博弈**：深度心理博弈与道德困境
5. **东方美学叙事**：中式审美与叙事风格
6. **名场面**：高传播度的经典场景设计
7. **修仙2.0**：传统修仙的创新变体
8. **短剧改编**：为短剧化设计的网文

### AI辅助写作陷阱

1. **剧情雷同**：AI倾向于生成相似剧情
2. **角色扁平**：AI角色缺乏深度和矛盾
3. **对话生硬**：AI对话缺乏口语感和个性
4. **逻辑过度完美**：缺乏真实世界的混沌感
5. **情感浅薄**：AI难以模拟复杂情感
6. **创意趋同**：AI在相同提示下产出相似内容
7. **设定堆砌**：AI倾向于堆砌设定而非有机融入
8. **节奏均质**：AI缺乏节奏变化的直觉

### 类型融合操作

1. 查阅 `novel-genres/references/genre-fusion-matrix.md` 获取相容性
2. 选择主题材+副题材
3. 找到融合接口
4. 设计融合规则
5. 验证自洽性
6. 小范围试写

### 创新落地模板

当用户要“给3个新点子”时，每个点子按这个格式输出：

| 字段 | 内容 |
|---|---|
| 熟悉入口 | 读者一眼能懂的旧套路或类型承诺 |
| 反转点 | 真正的新意，只改一个核心变量 |
| 爽点保留 | 读者原本期待的奖励如何兑现 |
| 风险 | 可能劝退读者的陌生点或理解成本 |
| 补救 | 用角色目标、开篇冲突或金手指规则降低风险 |
| 首场证明 | 前1000字里能证明这个点子成立的一场戏 |

当用户要“题材融合”时，按这个格式输出：主题材、副题材、融合接口、不能融合的部分、前三章兑现、长期升级空间。融合失败时优先删副题材，不要牺牲主题材承诺。

### Resource Routing

创新任务先判断该查哪个资料，不要在当前 skill 里硬编全部细节。

| 任务 | 优先资源 | 使用方式 |
|---|---|---|
| 反套路 | `references/anti-cliche-library.md` | 找旧套路，再只反转一个核心变量 |
| 题材融合 | `references/genre-fusion-compatibility.md` + `novel-genres/references/genre-fusion-matrix.md` | 先判相容性，再定主题材和副题材 |
| AI陷阱规避 | `references/ai-writing-pitfalls.md` | 检查雷同、扁平、对话生硬和节奏均质 |
| 热梗植入 | `references/meme-implantation.md` | 判断梗是否服务角色或冲突 |
| 平台适配 | `novel-commercialization` | 只在用户给出目标平台时调用 |

输出时注明“已用资源方向”，例如：已按题材融合矩阵判断主副类型相容，再给桥段。不要把资源名堆给用户而不落实到方案。

## Execution Protocol

1. 先确认用户要创新的是题材、金手指、人物关系、叙事结构还是卖点包装。
2. 每个创新点都必须绑定一个熟悉的读者入口，避免只新不爽、只怪不读。
3. 输出时给“原套路、反套路改法、风险、补救方式”四项。
4. 如果用户目标平台明确，优先匹配该平台读者接受度。

### Failure Handling

| 触发条件 | 处理动作 |
|---|---|
| 创新点过于陌生 | 增加一个经典爽点或熟悉题材作为锚点 |
| 反套路削弱爽感 | 保留情绪回报，只反转过程不取消奖励 |
| 融合题材互相打架 | 明确主题材和副题材，删掉第三个冲突元素 |
| 用户只要脑洞 | 先给3个可选方向，不展开完整大纲 |

### CHECKPOINT

最终输出前确认：创新点一句话能讲清、读者入口明确、资源路由正确、首场证明具体、爽点没有被反套路抵消。

### Anti-Patterns

- 不要为了新奇牺牲可读性。
- 不要把多个热门元素机械拼贴。
- 不要让反套路变成反读者期待。
- 不要只给概念，不给落地桥段。
