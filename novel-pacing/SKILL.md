# 节奏与结构工程

## Purpose

提供网文节奏控制的完整方法论，从开篇300字抓住读者到完本节奏收束，确保读者从第一句到最后一章都保持阅读欲望。

## When to Use

- 用户需要设计或优化开篇
- 用户需要规划章节结构与断章点
- 用户需要编排高潮与爽点节奏
- 用户需要控制字数与信息密度
- 用户遇到节奏拖沓/过快问题

## Instructions

### 开篇设计：3秒定输赢

当前网文读者的耐心极度有限，受短视频/短剧影响，开篇300字必须抓住读者。

**开篇公式**：强冲突 + 小钩子 + 大钩子 + 卡点钩子

**黄金三章核心任务**：
- 第一章：认识主角 → 建立共情 → 抛出核心矛盾
- 第二章：深化矛盾 → 展现金手指/特色 → 小爽点
- 第三章：定调子 → 显亮点 → 大钩子/断章

**四种钩子类型**：
1. 悬念钩（反常）：不合理现象/反常行为/诡异事件
2. 情绪钩（愤怒/不公）：主角被欺压/不公待遇/激发愤怒
3. 利益钩（贪婪/金手指）：获得宝物/系统激活/奇遇降临
4. 反差钩（猎奇/反套路）：打破常规/意外反转/认知颠覆

### 高潮编排

**高潮密度控制**：
- 小高潮：每2-3章一个
- 中高潮：每8-10章一个
- 大高潮：每卷（30-50章）一个
- 终极高潮：全书一个

**高潮结构**：
1. 铺垫蓄力（40%）：矛盾积累、压力递增
2. 爆发点（20%）：冲突达到顶点
3. 高潮对决（25%）：正面对决/真相揭露
4. 余波收束（15%）：结果、影响、新平衡

### 字数节奏曲线

| 阶段 | 字数范围 | 节奏特征 |
|------|---------|---------|
| 开篇期 | 1-10章 | 快节奏、高密度、强钩子 |
| 发展期 | 11-50章 | 中速推进、信息展开、关系建立 |
| 中盘期 | 51-150章 | 张弛有度、多条线交织、高潮迭起 |
| 后期 | 151-300章 | 加速推进、收线合拢、高潮密集 |
| 终局 | 300+章 | 全面爆发、终极对决、收束完本 |

### 多幕式结构

**三幕式**（短篇/卷）：
- 第一幕：设定与触发（25%）
- 第二幕：对抗与成长（50%）
- 第三幕：决战与解决（25%）

**五幕式**（长篇）：
- 第一幕：世界与主角介绍
- 第二幕：冲突升级
- 第三幕：中点转折
- 第四幕：危机深化
- 第五幕：终极对决与解决

### 章节节奏诊断表

诊断单章或开篇拖沓时，按表格输出。任一行为空，说明该段落需要删除、压缩或移动。

| 段落/场景 | 剧情功能 | 冲突 | 新信息 | 情绪转折 | 下一钩子理由 | 处理 |
|---|---|---|---|---|---|---|
| 开场 | 读者为什么进入本章 | 谁阻止谁 | 新出现的问题或目标 | 情绪从什么变到什么 | 为什么要继续读 | 保留/压缩/后移/删除 |
| 中段 | 推动局势变化 | 压力如何升级 | 新线索或代价 | 期待如何变化 | 是否制造下个选择 | 保留/压缩/后移/删除 |
| 结尾 | 停在什么不可回头点 | 危机或选择是否明确 | 揭露了什么 | 情绪是否未完成 | 下一章第一幕能接什么 | 保留/重写 |

输出修改方案时，不要只说“加快节奏”。必须明确哪一段删、哪一段压、哪条信息后移、结尾补哪种钩子。

## Execution Protocol

1. 先判断问题出在开篇、单章、卷结构、高潮还是断章。
2. 节奏诊断必须看目标读者、平台字数习惯和当前剧情功能。
3. 修改建议按“删、压、移、补钩子”四类给出。
4. 输出结构方案时标明每段字数占比和情绪变化。

### Failure Handling

| 触发条件 | 处理动作 |
|---|---|
| 开篇慢热 | 前置冲突、危险、欲望或异常事件 |
| 中段拖沓 | 合并重复场景，删除不改变局势的对话 |
| 高潮无力 | 补足代价、倒计时和不可逆后果 |
| 断章无钩子 | 在信息揭露、选择或危机升级处停章 |

### CHECKPOINT

最终输出前确认：每一段都有剧情功能、每章结尾有继续阅读理由、高潮前有足够蓄力，删改方案能落到具体段落。

### Anti-Patterns

- 不要把快节奏误解为只删描写。
- 不要每章都用同一种悬念断章。
- 不要让铺垫长期没有回报。
- 不要为了制造钩子强行中断自然场景。
