# 世界观与设定体系

## Purpose

提供网文世界观构建的完整方法论，从力量体系到地图格局，从经济体系到设定一致性维护，确保世界观宏大而自洽。

## When to Use

- 用户需要构建或完善世界观
- 用户需要设计力量/升级体系
- 用户需要规划地图与势力格局
- 用户需要建立经济/社会体系
- 用户遇到设定前后矛盾问题

## Instructions

### 力量体系架构

#### 修仙类模板
- 境界：炼气→筑基→金丹→元婴→化神→渡劫→大乘→飞升
- 每境界分：初期→中期→后期→巅峰
- 突破条件：灵力积累+悟道+天劫
- 战力跨度：每大境界10倍战力差

#### 玄幻类模板
- 等级：1-9阶或自定义命名
- 每阶分：初阶→中阶→高阶→巅峰
- 突破条件：修炼+机缘+战斗
- 特殊体系：异火/灵兽/血脉/天赋

#### 科幻类模板
- 文明等级：卡尔达肖夫I/II/III型
- 个人战力：基因等级/机甲等级/异能等级
- 科技树：材料→能源→信息→生物→空间

#### 都市类模板
- 异能等级：F→E→D→C→B→A→S→SS→SSS
- 组织架构：官方（异能局）+民间（世家/门派）
- 隐世规则：超凡世界对普通人隐藏

### 地图与势力格局

#### 地图扩展原则
1. 初期：小范围（村庄/学院/城市）
2. 中期：中范围（国家/大陆/星域）
3. 后期：大范围（世界/宇宙/多元）
4. 每次扩展都要有合理过渡

#### 势力设计模板
- 宗门/组织：3-5个主要势力+若干次要势力
- 势力关系：联盟/敌对/中立/暗斗
- 势力特色：每个势力有独特文化/功法/目标
- 势力平衡：不能一家独大，也不能完全均势

### 经济体系

#### 修仙经济
- 货币：灵石（下/中/上/极品）
- 资源：灵草/丹药/法器/功法
- 获取：任务/探索/炼制/交易
- 消耗：修炼/战斗/突破

#### 都市经济
- 货币：人民币+特殊货币
- 资源：异能材料/古董/灵物
- 获取：任务/商业/势力分配
- 消耗：修炼/装备/情报

### 设定一致性维护

#### 设定文档管理
1. 建立设定文档，记录所有已出场设定
2. 每次新增设定检查与旧设定是否矛盾
3. 定期回顾设定文档，清理冲突
4. 使用编号/标签系统管理设定

#### 常见设定崩坏
1. 战力崩坏：前期强敌后期弱鸡
2. 设定遗忘：前面提到的设定后面忘了
3. 逻辑矛盾：同一设定前后说法不同
4. 补丁过多：不断打补丁解释矛盾

#### 新设定输出 Schema

新增任何力量、地图、势力、经济或谜题设定时，按以下格式输出。缺任一项就先补齐，不要直接堆名词。

| 字段 | 必填内容 | 作用 |
|---|---|---|
| 规则 | 这个设定如何运行，谁能使用，触发条件是什么 | 防止只有概念没有机制 |
| 代价 | 使用它会消耗什么，失败会损失什么 | 防止主角无成本变强 |
| 限制 | 不能做什么，在哪些环境下失效 | 防止万能设定 |
| 例外 | 谁可以绕过限制，为什么合理 | 给反转和高级角色留空间 |
| 剧情用途 | 当前30章内会怎样推动冲突 | 防止世界观脱离剧情 |
| 升级路径 | 中后期如何扩展，扩展后旧设定是否仍有效 | 防止后期吃掉前期规则 |
| 冲突钩子 | 哪个角色或势力会因它产生利益冲突 | 把设定变成剧情发动机 |

示例：都市异能“记忆回声”不是一句“能读取记忆”。要写成：接触物品可读取最近24小时残留情绪；每次使用消耗睡眠并造成偏头痛；不能读取主动撒谎者的真实意图；少数训练者可伪造残留；前三十章用于追查失踪案；后期升级为读取地点创伤；异能局和黑市鉴定师都会争夺主角。

### 悬念与谜题铺设

1. **世界级悬念**：世界的终极真相（贯穿全书）
2. **势力级悬念**：某势力的秘密（跨卷）
3. **人物级悬念**：某角色的真实身份/目的（跨章）
4. **事件级悬念**：某事件的真相（单卷/数章）

## Execution Protocol

1. 先确认题材、主角层级、故事规模和目标平台，再设计世界观。
2. 每次新增设定都写清楚“规则、代价、限制、例外”，避免只有酷炫名词。
3. 输出优先给可直接放进设定文档的表格或条目。
4. 如果用户已有设定，先做一致性检查，再补新内容。

### Workflow Branches

| 用户输入 | 先做什么 | 输出重点 |
|---|---|---|
| 只有题材 | 给3个世界观方向，让用户选规模和调性 | 不展开全套设定，只给入口冲突 |
| 已有力量体系 | 查规则、代价、限制、例外是否完整 | 修补漏洞和升级路径 |
| 已有地图势力 | 查势力目标、资源和冲突关系 | 让势力推动剧情，而不是背景板 |
| 设定前后冲突 | 列旧设定、新设定、冲突点、影响章节 | 给保留旧设定或重写新设定方案 |
| 要长篇扩展 | 先锁当前30章，再设计中后期升级 | 保证扩展不废掉前期规则 |

如果用户没有给字数规模，默认按“当前30章可用”设计，不直接铺到全书终局。

### Failure Handling

| 触发条件 | 处理动作 |
|---|---|
| 用户只给模糊题材 | 先给3个方向让用户选，不直接铺满完整世界 |
| 新设定与旧设定冲突 | 标出冲突点，给出保留旧设定或重写新设定两种方案 |
| 战力体系膨胀过快 | 增加代价、冷却、资源消耗或境界门槛 |
| 世界观压过剧情 | 收缩设定，只保留当前30章会用到的内容 |

### CHECKPOINT

最终输出前确认：已选择正确流程分支、主角成长路径清楚、力量有代价、限制和例外明确、地图扩展有理由、势力关系能制造剧情冲突、当前30章内用得上。

### Anti-Patterns

- 不要堆名词表而不给规则。
- 不要让后期设定直接废掉前期冲突。
- 不要把所有势力写成同一种宗门换皮。
- 不要一次性展开远超当前剧情需要的宇宙级设定。
