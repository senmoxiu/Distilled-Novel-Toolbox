# 商业化与平台特化

## Purpose

提供网文商业化全链路指南，帮助作者选择最适合的平台，了解各平台风格与规则，设计最优变现策略。

## When to Use

- 用户需要选择发布平台
- 用户需要了解某平台的风格与规则
- 用户需要设计变现模式
- 用户需要规划签约与收入策略

## Instructions

### 平台全景对比

#### 起点中文网
- 定位：最大的付费阅读平台
- 读者画像：男性为主，18-35岁，付费意愿强
- 适合题材：玄幻、仙侠、都市、科幻、历史
- 风格要求：有一定文学性，剧情有深度
- 签约门槛：中等，需1-3万字内投
- 收入模式：订阅+打赏+月票+道具
- 优势：品牌效应强、IP开发渠道多、创作体系完善
- 劣势：竞争激烈、新人出头难

#### 番茄小说
- 定位：免费阅读平台（背靠抖音/字节）
- 读者画像：泛读者，年龄跨度大，免费阅读
- 适合题材：爽文、短平快、都市、言情
- 风格要求：节奏快、爽感密、信息直给
- 签约门槛：较低
- 收入模式：广告分成+全勤（600元/月）
- 优势：流量优势明显、入行门槛低
- 劣势：收入不透明、单本收入上限低

#### 飞卢小说网
- 定位：脑洞/同人/快节奏平台
- 读者画像：年轻男性，追求爽感
- 适合题材：同人、诸天、脑洞、系统流
- 风格要求：爽感十足、脑洞大、节奏极快
- 签约门槛：低（8千字可内投）
- 收入模式：订阅+打赏
- 优势：门槛低、更新快、变现快
- 劣势：更新压力极大、作品寿命短

#### 晋江文学城
- 定位：女性向阅读平台
- 读者画像：女性占比90%+，18-30岁
- 适合题材：言情、耽美、大女主、古言
- 风格要求：情感细腻、文笔好、人设精致
- 签约门槛：中等
- 收入模式：订阅+打赏+定制
- 优势：女性向第一平台、IP价值高
- 劣势：男性向题材不吃香、审核严格

#### 七猫小说
- 定位：免费阅读平台
- 读者画像：下沉市场，年龄偏大
- 适合题材：赘婿、神医、兵王、都市
- 风格要求：传统爽文、节奏快
- 收入模式：广告分成+全勤
- 优势：下沉市场流量大
- 劣势：题材受限、收入上限低

### 付费 vs 免费核心区别

| 维度 | 付费平台 | 免费平台 |
|------|---------|---------|
| 收入透明度 | 高（卖出多少清楚可算） | 低（广告分成难以验证） |
| 收入上限 | 高（爆款月入百万） | 低（多数几千到几万） |
| 读者质量 | 高（付费=认真读） | 低（泛读者，跳出率高） |
| 写作要求 | 高（需有深度和逻辑） | 低（爽就完事） |
| 竞争强度 | 高 | 中 |
| 适合新人 | 有写作基础的新人 | 零基础新人 |

### 变现设计

1. **订阅收入**：核心收入来源
2. **打赏收入**：粉丝经济
3. **月票/推荐票**：排名与曝光
4. **全勤奖**：稳定更新奖励
5. **IP改编**：影视/动漫/游戏
6. **版权运营**：出版/海外/衍生

### 数据诊断框架

当用户反馈“追读低、收藏还行、完读差、点击高但没人看下去”时，先按数据漏斗定位，不要直接建议重写全书。

| 指标 | 主要判断 | 常见问题 | 处理动作 |
|---|---|---|---|
| 曝光 | 平台是否给到流量 | 题材标签、简介、封面不匹配 | 调整卖点包装和标签，不急着改正文 |
| 点击 | 读者是否愿意打开 | 标题、简介、封面承诺不够强 | 强化一句话卖点和前三章冲突承诺 |
| 收藏 | 读者是否认为值得留存 | 设定有兴趣但角色目标不清 | 补主角欲望、金手指边界和长期期待 |
| 完读 | 单章是否能读完 | 注水、开场慢、信息解释太多 | 删除不改变局势的段落，冲突前置 |
| 追读 | 读者是否愿意回来 | 断章弱、爽点兑现慢、阶段目标断裂 | 重做章尾钩子和下一章即时期待 |
| 章评 | 读者在哪些点有反应 | 情绪点缺失或争议集中 | 保留有效情绪点，修复高频吐槽处 |

止损规则：连续7天追读无改善，且开篇30章已完成两轮针对性修改，停止局部修补，改做新书复盘。不要为了沉没成本无限重写。

诊断输出格式：数据缺口、最可能问题、证据、3个改法、观察周期、止损条件。

## Execution Protocol

1. 先确认用户目标：签约、变现、选平台、改开篇、数据复盘或多平台分发。
2. 平台建议必须结合题材、更新能力、字数规划和作者承受风险。
3. 商业化方案输出时给出投入、回报周期、关键指标和止损条件。
4. 如果用户已有作品，先看开篇卖点和目标读者，再谈平台。

### Goal Branches

| 用户目标 | 先问什么 | 输出重点 |
|---|---|---|
| 选平台 | 题材、预计字数、更新速度、风险偏好 | 平台匹配、改包装建议、首发策略 |
| 签约 | 已写字数、开篇卖点、目标编辑或渠道 | 内投材料、前三章修法、等待周期 |
| 变现 | 可投入时间、是否能稳定更新、是否接受长周期 | 收益路径、成本、关键指标 |
| 数据复盘 | 曝光、点击、收藏、完读、追读、章评 | 漏斗诊断、观察周期、止损条件 |
| 多平台分发 | 版权状态、首发限制、作品类型 | 主平台、副平台、风险提示 |

如果用户只说“想赚钱”，先走选平台分支，再补变现分支；不要直接推荐单一平台。

### Failure Handling

| 触发条件 | 处理动作 |
|---|---|
| 用户只问哪个平台赚钱 | 先按题材、速度、经验和风险偏好筛选 |
| 数据不足 | 明确缺口，给可收集的数据清单 |
| 平台规则可能变化 | 标注需复核，不给绝对承诺 |
| 作品与平台错配 | 给改包装或换平台两套方案 |

### CHECKPOINT

最终输出前确认：已选择正确目标分支、推荐平台有理由、收益预期不过度夸大、数据诊断有证据、方案包含可执行下一步和止损条件。

### Anti-Patterns

- 不要承诺签约或收入结果。
- 不要把所有题材都推荐到同一个平台。
- 不要忽视作者更新能力。
- 不要只谈流量，不谈完读、追读和转化。
